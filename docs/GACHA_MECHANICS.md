# Механика гачи — Winx Gacha

## Ежедневные прокрутки

- Игрок получает **10 прокруток в день**, максимальный накопленный запас — **40**
- Дата последнего пополнения хранится в `localStorage`; при смене даты запас пополняется на 10 (до лимита 40)
- Игровой валюты нет — только счётчик прокруток

## Редкости айтемов

| Редкость | Тип айтема | Hard pity | Значок |
|----------|------------|-----------|--------|
| Common | Обычный предмет | — | ⚪ |
| Rare | Элемент одежды / аксессуар | каждые **10** прокруток | 🔵 |
| Legendary | Персонаж Winx | каждые **120** прокруток | 🟡 |

## Базовые вероятности (до soft pity)

| Редкость | Вероятность |
|----------|-------------|
| Legendary | 0.6% |
| Rare | 5.1% |
| Common | 94.3% |

## Soft pity

### Для Rare (пуллы 7–10)

```
P_rare(n) = 0.051,               n < 7
P_rare(n) = 0.051 + 0.20*(n-6),  7 ≤ n < 10
P_rare(10) = 1.0                  (hard pity)
```

Пример роста:
| Пулл без Rare | Вероятность Rare |
|---------------|-----------------|
| 1–6 | 5.1% |
| 7 | 25.1% |
| 8 | 45.1% |
| 9 | 65.1% |
| 10 | 100% |

### Для Legendary (пуллы 90–120)

```
P_leg(n) = 0.006,                n < 90
P_leg(n) = 0.006 + 0.066*(n-89), 90 ≤ n < 120
P_leg(120) = 1.0                  (hard pity)
```

Пример роста:
| Пулл без Legendary | Вероятность Legendary |
|--------------------|-----------------------|
| 1–89 | 0.6% |
| 90 | 7.2% |
| 100 | 73.8% |
| 110 | ~100% |
| 120 | 100% |

## Алгоритм одного пулла

```
function pull(gameState):
  gameState.pityRare++
  gameState.pityLegendary++

  // Hard pity (высший приоритет)
  if pityLegendary >= 120:
    item = pickLegendary()
    reset pityRare = 0, pityLegendary = 0
    return item

  if pityRare >= 10:
    item = pickRare()
    reset pityRare = 0
    return item

  // Soft pity
  legRate = computeLegRate(pityLegendary)
  rareRate = computeRareRate(pityRare)

  roll = Math.random()

  if roll < legRate:
    reset pityRare = 0, pityLegendary = 0
    return pickLegendary()

  if roll < legRate + rareRate:
    reset pityRare = 0
    return pickRare()

  return pickCommon()
```

### Правила сброса счётчиков pity

- `pityRare` сбрасывается при выпадении **Rare или Legendary**
- `pityLegendary` сбрасывается только при выпадении **Legendary**
- Оба счётчика **персистентны** между сессиями (хранятся в localStorage)

## 10-кратный пулл (x10)

- Выполняется 10 последовательных одиночных пуллов с единым счётчиком pity
- Гарантия x10: если ни в одном из 10 пуллов не выпало Rare или Legendary — последний пулл форсируется как Rare
- Потраченные прокрутки: 10 за раз

## Дубликаты

| Ситуация | Результат |
|----------|-----------|
| Дубликат Legendary (персонаж уже открыт) | Выдаётся «Звёздочка силы» — редкий предмет кастомизации |
| Дубликат Rare / Common | Добавляется в инвентарь ещё один экземпляр (без ограничений) |

## Выбор айтема при выпадении

- **pickLegendary()** — равновероятный случайный выбор из пула всех персонажей (`CHARACTERS`)
- **pickRare()** — равновероятный случайный выбор из пула нарядов и аксессуаров (`OUTFITS + ACCESSORIES`)
- **pickCommon()** — равновероятный случайный выбор из пула обычных предметов (`COMMONS`)

## Худшие сценарии (worst case)

| Событие | Максимум пуллов |
|---------|-----------------|
| Следующий Rare | 10 пуллов |
| Следующий Legendary | 120 пуллов |
| Конкретный Legendary (при 14 персонажах) | 120 × 14 = 1680 пуллов |

## Ограничение прокруток

- При попытке открыть кейс без прокруток — действие заблокировано (кнопка disabled)
- Кнопка x10 заблокирована при `pullsAvailable < 10`
